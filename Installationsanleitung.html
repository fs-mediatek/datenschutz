🛡 Datenschutz-Tool – Installationsanleitung

DSGVO Compliance Manager · Ubuntu Server / Desktop · Stand: März 2026

1 Voraussetzungen

Betriebssystem Ubuntu 22.04 LTS oder 24.04 LTS
Node.js Version 18 oder höher (empfohlen: 20 LTS)
Git Zum Klonen des Repositories
Speicherplatz Min. 500 MB (inkl. node_modules + Build)

2 System aktualisieren

# System-Pakete aktualisieren
sudo apt update && sudo apt upgrade -y

3 Node.js 20 LTS installieren

Das offizielle NodeSource-Repository verwenden:

# NodeSource-Repository hinzufügen
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# Node.js installieren
sudo apt install -y nodejs

# Version prüfen
node --version
# Erwartete Ausgabe: v20.x.x

npm --version
# Erwartete Ausgabe: 10.x.x

Alternativ: Falls Node.js bereits über snap oder nvm installiert ist, reicht Version 18+.

4 Git installieren

sudo apt install -y git

# Prüfen
git --version
# Erwartete Ausgabe: git version 2.x.x

5 Repository klonen

# In gewünschtes Verzeichnis wechseln
cd /opt

# Repository klonen (URL anpassen!)
sudo git clone https://github.com/BENUTZERNAME/datenschutz-tool.git

# Berechtigungen setzen
sudo chown -R $USER:$USER /opt/datenschutz-tool

# In das Verzeichnis wechseln
cd /opt/datenschutz-tool

Hinweis: Die Repository-URL https://github.com/BENUTZERNAME/datenschutz-tool.git muss durch die tatsächliche URL ersetzt werden.

6 Automatische Installation

Option A: Automatischer Installer (empfohlen)

# Installer-Skript ausführbar machen und starten
chmod +x github-install.sh
./github-install.sh --dir /opt/datenschutz-tool

Der Installer führt automatisch alle Schritte aus:

  1. Prüft Node.js und Git
  2. Installiert npm-Abhängigkeiten
  3. Fragt Port und URL ab
  4. Richtet die SQLite-Datenbank ein
  5. Lädt DSGVO-Stammdaten (Rechtsgrundlagen, Datenkategorien, TOMs)
  6. Erstellt den Produktions-Build
  7. Bietet Systemd-Service-Einrichtung an

Option B: Manuelle Installation

# 1. Abhängigkeiten installieren
npm install

# 2. Umgebungsdatei erstellen
cp .env.example .env

# 3. Prisma Client generieren
npx prisma generate

# 4. Datenbank-Migrationen ausführen
npx prisma migrate deploy

# 5. DSGVO-Stammdaten laden (optional, aber empfohlen)
npx tsx prisma/seed.ts

# 6. Produktions-Build erstellen
npx next build

7 Anwendung starten

Manuell starten (Vordergrund)

npx next start -p 3000

# Ausgabe:
#   ▶ Ready on http://localhost:3000

Mit Start-Skript (Auto-Restart)

chmod +x start.sh
./start.sh

Geschafft! Die Anwendung ist jetzt erreichbar unter http://SERVER-IP:3000

8 Systemd-Service einrichten (Produktionsbetrieb)

Für den Dauerbetrieb empfiehlt sich ein Systemd-Service, der die Anwendung automatisch startet und bei Abstürzen neu startet.

# Service-Datei erstellen
sudo nano /etc/systemd/system/datenschutz-tool.service

Folgenden Inhalt einfügen:

[Unit]
Description=Datenschutz-Tool DSGVO Compliance
After=network.target

[Service]
Type=simple
User=ihr-benutzername
WorkingDirectory=/opt/datenschutz-tool
ExecStart=/usr/bin/npx next start -p 3000
Restart=on-failure
RestartSec=5
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
# Service aktivieren und starten
sudo systemctl daemon-reload
sudo systemctl enable datenschutz-tool
sudo systemctl start datenschutz-tool

# Status prüfen
sudo systemctl status datenschutz-tool

9 Nginx Reverse Proxy (optional)

Für den Zugriff über Port 80/443 mit eigenem Domainnamen:

# Nginx installieren
sudo apt install -y nginx

# Konfiguration erstellen
sudo nano /etc/nginx/sites-available/datenschutz-tool

Konfigurationsinhalt:

server {
    listen 80;
    server_name datenschutz.ihre-domain.de;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
# Seite aktivieren und Nginx neu laden
sudo ln -s /etc/nginx/sites-available/datenschutz-tool /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# Optional: SSL-Zertifikat mit Let's Encrypt
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d datenschutz.ihre-domain.de

10 Firewall konfigurieren

# Falls UFW aktiv ist – Port freigeben
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Oder direkt Port 3000 (ohne Nginx)
sudo ufw allow 3000/tcp

# Status prüfen
sudo ufw status

11 Update durchführen

cd /opt/datenschutz-tool

# Neueste Version holen
git pull

# Abhängigkeiten aktualisieren
npm install

# Datenbank-Migrationen anwenden
npx prisma generate
npx prisma migrate deploy

# Neu bauen und Service neu starten
npx next build
sudo systemctl restart datenschutz-tool

12 Backup

Die gesamte Datenbank liegt in einer einzigen SQLite-Datei:

# Datenbank sichern
cp /opt/datenschutz-tool/dev.db /opt/datenschutz-tool/backup_$(date +%Y%m%d).db

# Oder als Cronjob (täglich um 2:00 Uhr)
sudo crontab -e
# Folgende Zeile hinzufügen:
0 2 * * * cp /opt/datenschutz-tool/dev.db /opt/backups/datenschutz_$(date +\%Y\%m\%d).db

Wichtig: Die Datei dev.db enthält alle Unternehmensdaten, Verarbeitungsverzeichnisse und Datenlandkarten. Regelmäßige Backups sind dringend empfohlen!

Fehlerbehebung

ProblemLösung
Port 3000 bereits belegt sudo lsof -i :3000 – Prozess identifizieren und Port in .env ändern
Permission denied bei npm install sudo chown -R $USER:$USER /opt/datenschutz-tool
Prisma-Fehler nach Update npx prisma generate && npx prisma migrate deploy
Leere Seite im Browser Build prüfen: npx next build – auf Fehlermeldungen achten
Service startet nicht sudo journalctl -u datenschutz-tool -n 50 für Fehlermeldungen
Umlaute fehlerhaft Locale prüfen: locale – sollte UTF-8 enthalten
